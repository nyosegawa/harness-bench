# Architecture Decision Records

This directory records material design decisions for HarnessBench.

ADRs use a lightweight Markdown format with YAML frontmatter:

- `id`: stable decision id
- `title`: short decision title
- `status`: `proposed`, `accepted`, `superseded`, or `rejected`
- `date`: decision date
- `deciders`: people or roles involved
- `tags`: searchable decision categories
- `related`: related ADRs or docs

Each ADR should explain the context, the decision, consequences, alternatives,
and follow-up conditions. ADRs are append-only decision records; supersede a
decision with a new ADR rather than rewriting its history.
