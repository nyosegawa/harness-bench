Executive Summary

This baseline compares three memory-disabled agent harnesses on the same 27 real-repository debugging cases. Codex and Claude each pass 18/27 cases, Cursor passes 17/27. Claude has the lowest observed cost per pass in this run, Cursor has the fastest median wall time, and Codex is roughly tied with Claude on hidden-oracle pass rate while using more estimated tokens and cost.

The headline score is a hidden-oracle pass rate, not yet the final correctness score. False-negative review found several cases where hidden tests may over-constrain implementation details or unstated API choices; those cases are called out below and should be resolved before using the matrix as a final leaderboard.

Benchmark Design

Each case starts from a real repository base commit where the hidden core test fails, and a fixed commit where the same hidden test passes. Agent runs receive only the issue-style instruction and work inside an isolated checkout. A run is counted as pass only when the hidden core test passes after the agent edit. Raw harness logs are kept locally, metrics are normalized per harness, and invalid infrastructure runs are excluded from baseline summaries.

The case set spans 9 repositories with low, mid, and high difficulty tasks per repository. Difficulty is based on expected debugging complexity, not just repository size. Repo size is tracked separately because a large repository can still have a localized bug, while a small repository can require subtle behavior reconstruction.

Turns, tokens, tool calls, and cost are intentionally not collapsed into one ambiguous number. Codex turns mean completed harness invocations, Claude turns and cost come from Claude CLI output, and Cursor usage is normalized from stream events. Cached input is separated from fresh input where the harness exposes it; Codex and Cursor dollar values are API-equivalent estimates, while Claude cost is reported by the harness.

Case Catalog

axios-axios-high-http-connect-timeout

high small axios/axios

Node HTTP requests with maxRedirects set to 0 do not honor the configured timeout while the TCP connection is still pending. Reproduce the hanging connect phase and make timeout handling abort the request consistently without changing timeout: 0 behavior.

Higher-difficulty networking lifecycle bug involving native ClientRequest timeout semantics, abort handling, and maxRedirects transport selection.

axios-axios-low-settle-error-code

low small axios/axios

Custom validateStatus callbacks can reject statuses outside the normal 4xx and 5xx ranges, and AxiosError.code becomes undefined. Reproduce the settle behavior and ensure rejected responses always receive a defined bad-request or bad-response error code.

Focused core error-classification bug with a narrow implementation surface. Hidden tests call settle directly and require codes for custom rejected non-4xx/5xx statuses.

axios-axios-mid-fetch-global-access

mid small axios/axios

The fetch adapter can throw during module load when the internal global reference is temporarily unavailable. Reproduce the module-load failure and defer global fetch API access so adapter creation still works with explicit env overrides.

Adapter initialization bug that requires understanding module evaluation and explicit environment overrides. Hidden tests avoid browser infrastructure and check the module-load edge case directly.

fastapi-fastapi-high-pydantic-json-fast-path

high large fastapi/fastapi

FastAPI responses with a Pydantic return type or response model still go through the slower Python json.dumps path when the default response class is used. Reproduce response serialization for Pydantic models and use the direct Pydantic JSON bytes path only when no explicit response class overrides it.

Higher-difficulty routing/serialization behavior with a performance-sensitive fast path. Hidden test verifies both the optimized default path and the explicit JSONResponse fallback.

fastapi-fastapi-low-remove-vibe-decorator

low large fastapi/fastapi

A temporary April Fools API decorator was left on FastAPI instances. Remove that public decorator and its dedicated behavior without affecting normal HTTP and websocket route registration.

Low-difficulty public API cleanup. Hidden tests assert the temporary decorator is gone while ordinary route decorators continue to work.

fastapi-fastapi-mid-jsonable-encoder-color-types

mid large fastapi/fastapi

FastAPI's JSON encoder still relies on deprecated Pydantic color imports and misses the newer pydantic-extra-types color class. Reproduce color encoding with current Pydantic v2 packages and support both legacy and new color classes without import-time failures.

Mid-difficulty compatibility bug around Pydantic v2 deprecations and optional extra color types. Hidden test exercises both import paths directly.

go-gitea-gitea-high-compare-no-common-history

high large go-gitea/gitea

Repository compare views break when two branches have no common history. Reproduce the Git comparison edge case and make the compare path handle unrelated histories gracefully instead of treating the missing merge base as an unexpected command failure.

Higher-difficulty Git compare failure where the symptom is a web compare workflow but the root cause is merge-base error semantics for unrelated branch histories.

go-gitea-gitea-low-schedule-null-payload

low large go-gitea/gitea

Scheduled Actions can panic when the stored event payload is JSON null. Reproduce the schedule payload handling edge case and make scheduled runs add the schedule field without crashing or dropping already valid payload fields.

Focused scheduled-action payload bug with a narrow helper-level failure. Hidden tests cover JSON null, empty payloads, and preserving existing payload keys when adding the schedule value.

go-gitea-gitea-mid-pr-merge-self-reference

mid large go-gitea/gitea

Pull request merge commits often mention the merged PR number in their commit message. Reproduce the issue timeline behavior and prevent a merged PR from creating a commit-reference comment to itself, while keeping normal issue and pull request references working.

Mid-difficulty issue timeline bug requiring fixture-backed model behavior: skip only the PR merge commit's reference to its own PR, without suppressing unrelated commit references.

jesseduffield-lazygit-high-branch-divergence-fast-path

high small jesseduffield/lazygit

Loading divergence from configured base branches is too slow in repositories with many local branches because each branch triggers separate Git commands. Reproduce branch behind-base calculation and use Git's ahead-behind ref formatting when available, while preserving the legacy behavior on older Git versions and clearing stale values for branches omitted from fresh output.

Higher-difficulty performance bug requiring knowledge of Git version-gated ahead/behind formatting, multiple base branches, parser robustness, and stale branch state handling.

jesseduffield-lazygit-low-github-owner-casing

low small jesseduffield/lazygit

GitHub pull request integration fails to associate a local branch with its pull request when the repository owner casing differs between the PR API response and the configured remote URL. Reproduce the mismatch and make PR lookup handle owner casing consistently without changing branch-name matching.

Small Go logic bug with a focused unit-level behavior. The original PR added a regression test and changed only GitHub owner normalization.

jesseduffield-lazygit-mid-preserve-commit-message-whitespace

mid small jesseduffield/lazygit

When a commit message is remembered and restored, whitespace in the body is not preserved correctly. Reproduce the commit-message round trip and fix the message/description handling so intentional blank lines and indentation are kept, while co-author trailers are still appended cleanly.

Multi-file TUI commit-message handling bug. Hidden tests check observable string behavior around preserved body whitespace and co-author trailer insertion without requiring the original helper decomposition.

langflow-ai-langflow-high-lfx-stream-fallback

high large langflow-ai/langflow

lfx run can crash or return empty output when stream=true because streaming model output is persisted without a session/event manager to drain chunks. Reproduce the streaming handler path and fall back to a normal ainvoke result when lfx lacks the runtime context required for token streaming.

Higher-difficulty runtime bug in streaming model handling for CLI execution. Hidden test exercises the async handler with missing event-manager context and verifies it returns plain text without persisting an unconsumed stream.

langflow-ai-langflow-low-loguru-file-routing

low large langflow-ai/langflow

Custom components that log through Loguru do not write to Langflow's configured log file. Reproduce logging with a configured file target and route Loguru messages through the Langflow logger without duplicating stale file handlers.

Focused logging integration bug. Hidden test configures a temporary log file and verifies Loguru output reaches the same sink as Langflow logs.

langflow-ai-langflow-mid-mcp-connectable-inputs

mid large langflow-ai/langflow

MCP tool inputs generated from number, boolean, and JSON schema fields render as UI-only widgets instead of connectable handles. Reproduce schema-to-input conversion and make non-string fields advertise the correct connection input types while preserving their value coercion behavior.

Mid-difficulty schema conversion and coercion bug crossing MCP model schema, input metadata, and frontend handle semantics.

louislam-uptime-kuma-high-websocket-auth-options

high medium louislam/uptime-kuma

WebSocket monitors ignore custom headers and authentication options during the upgrade handshake. Reproduce option construction for websocket checks and support headers, basic auth, OAuth client credentials, mTLS, and handshake timeouts without breaking existing status-code behavior.

Higher-difficulty monitor behavior spanning websocket handshake options and authentication precedence. Hidden tests check options directly to avoid flaky network timing.

louislam-uptime-kuma-low-submillisecond-ping-chart

low medium louislam/uptime-kuma

Monitor graphs show gaps for successful TCP checks whose average ping rounds down to 0 ms. Reproduce the chart datapoint filtering behavior and make zero-valued ping samples render while still hiding missing ping values.

Focused frontend data filtering bug. Hidden test executes the component method against zero, null, and non-zero ping datapoints without launching a browser.

louislam-uptime-kuma-mid-uptime-cleanup-buckets

mid medium louislam/uptime-kuma

Uptime cleanup lookups can create missing historical buckets, which shifts the rolling data window and makes longer-period uptime views show too little recent data. Reproduce the missing-bucket side effect and prevent cleanup lookups from mutating minutely, hourly, or daily uptime queues.

Mid-difficulty state mutation bug in rolling uptime queues. Hidden tests avoid DB-heavy update paths and directly verify key lookup side effects.

sharkdp-bat-high-fallback-syntax

high small sharkdp/bat

Users need a way to specify a syntax-highlighting fallback that applies only when automatic syntax detection fails. Add support for such a fallback without changing explicit language selection or normal file-name based detection behavior.

Pilot-only high case. This is broader than a bug fix and behaves more like a missing capability, so it is useful for validating runner mechanics but may be replaced in the final benchmark by a high-difficulty bug-fix PR from another repository.

sharkdp-bat-low-zip-binary-detection

low small sharkdp/bat

ZIP archive files whose first bytes are a standard ZIP signature are being treated like text in header-only output. Reproduce the behavior and make bat classify ZIP archive content as binary, without changing the behavior for ordinary text that merely starts with similar characters.

Narrow CLI behavior bug with a small implementation surface and a simple external hidden test. The original PR also added unit and integration regressions.

sharkdp-bat-mid-control-character-wrapping

mid small sharkdp/bat

When binary content is intentionally rendered as text, lines containing control characters wrap incorrectly. Reproduce the wrapping problem with a narrow terminal width and fix the displayed-width calculation so the rendered output wraps at the expected column.

Terminal rendering bug that requires understanding how non-printable characters affect display width. The hidden test checks observable CLI wrapping behavior instead of the internal helper used by the PR.

Memo API conversions fail when related users have been deleted, such as reactions or comments whose creators no longer exist. Reproduce the failure and make memo listing and comment/reaction conversion tolerate missing related users without hiding valid memos from existing users.

Higher-difficulty API conversion bug spanning memo, comment, reaction, and user deletion paths. Hidden tests check observable API behavior across multiple service methods.
